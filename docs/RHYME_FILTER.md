# 古典語ハックフィルタ — なぜ必要か

## 一言で

**「韻は踏めるけど、現代の日本語として変な語」を候補から外す（または下位表示する）機能。**

MeCab とは別問題。韻 API（特に韻システム / nwnwn）が返す **辞書上の古典縮約形** が原因。

---

## 何が起きていたか

「大麻」「よかった」で韻検索すると、候補に例えば:

| 候補 | 実体 | 現代口語 |
|------|------|----------|
| 合間 | 普通の名詞 | ✅ 自然 |
| あんま | 普通の名詞 | ✅ 自然 |
| 白々し**きゃ** | 白々し**ければ** の縮約 | △ 文学・古い |
| 憎たらし**きゃ** | 憎たらし**ければ** | △ 同上 |
| **よだちゃ** | 意味不明に近い | ❌ |
| **かんばしりゃ** | 間走れば？ | ❌ |
| 殴り**とばしゃ** | 殴り飛ば**せば** | △ ラップなら通じることも |

韻システムは **母音パターンが合えば** こういう語も返す。  
以前のアプリは末尾「ゃ」に **加点** していたため、81% など高スコアで上位表示 → LLM が歌詞に採用。

---

## フィルタの意味（古典語ハック）

**古典語ハック** = 江戸期以降の文語・仮定形の縮約（〜ければ→〜きゃ、〜せば→〜しゃ）を、  
**韻のためだけに行末に置く** テクニック。

- ラップ・作詞の **探索素材** としては使える
- デフォルトの歌詞生成では **不自然になりやすい**

だから **デフォルト OFF**（候補から除外 + LLM に渡さない）。

---

## 使い分け

### デフォルト（古典韻 OFF）— おすすめ

Setup の「古典韻候補も含める」**チェックなし**

- 合間・あんま など自然な語が上位
- 歌詞も口語寄り
- 友人向け本番はこれ

### 古典韻 ON — 探索したいとき

Setup で **「古典韻候補も含める」** を ON

- 白々しきゃ 等も **韻タブに表示**（ラベル「古典韻」付き）
- スコアは **自然な語より下位**（ランキング修正済み）
- 意図的に古典韻を踏みたいバース向け

### サーバー全体で ON（通常不要）

Vercel 環境変数 `RHYME_ALLOW_ARCHAIC=true`  
→ 全ユーザーのデフォルトが ON（Setup チェックと併用可）

## MeCab との関係

| | 古典語ハックフィルタ | MeCab / kuromoji |
|--|---------------------|------------------|
| 目的 | 変な **韻候補** を減らす | **読み・分割・モーラ** を正確に |
| よだちゃ問題 | ✅ 直接効く | 効かない（韻 API 側の問題） |
| 読み仮名の精度 | — | ✅ 効く |
| Vercel | ✅ そのまま | kuromoji は可 / MeCab 本家は難 |

**両方あると理想** だが、今回の「変な日本語」対策の **本命はフィルタ**。

---

## 許容リスト（フィルタしない口語寄り）

意図的に残している例:

- 拝みゃ、窄まりゃ、伸ばしゃ、殴りとばしゃ（ラップ口語として通じる系）
- なきゃ、〜じゃ 等の日常縮約

リストは `rhymeWordValidity.ts` の `ALLOWED_COLLOQUIAL_RHYMES` で管理。
